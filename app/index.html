How to Use Yeoman


Using Yeoman

A better way to build a website

Today we will learn how to integrate Yeoman into your workflow.

First, what is Yeoman?

“Yeoman is an open source client-side development stack, consisting of tools and frameworks intended to help developers quickly build high quality web applications. Yeoman runs as a command-line interface written in Node.js which combines several functions into one place, such as generating a starter template, managing dependencies, running unit tests, providing a local development server, and optimizing production code for deployment.”

Wikipedia on Yeoman

Yeoman, in a nutshell, is three different tools: yo, bower, and grunt. These tools work together with the goal being of saving you, the programmer time. This happens by letting the computer do as much of the automation as possible for building your future website or webapp.

Yo is a frameworks tool that when launched, builds the skeleton of your future website. However, Yo alone just produces a nice-looking and ready-made website.

Bower then combs github and other repositories for the necessary libraries, and integrates them into your website.

Finally, grunt is used to support this whole process. Grunt gives you immediate access to a virtual server for live output, testing options through Karma, and added functionality to condense your code and files down for faster read/write.

So basically, Yeoman lets you select from a menu what type of program you want to make, then lets you select what kind of bells and whistles are going on the site, retrieves those bells and whistles for you, and then later tests your site to make sure said bells and whistles actually work.
